# Magmasystems Blog: 数据库架构师是做什么的？

> 来源：[`magmasystems.blogspot.com/2006/11/what-does-database-architect-do.html#0001-01-01`](http://magmasystems.blogspot.com/2006/11/what-does-database-architect-do.html#0001-01-01)

即使我曾是微软 SQL Server 团队的第一个开发者，我不得不承认数据库并不让我感到兴奋... 你必须有一种特殊的心态才能整天处理数据库，说实话，我的兴趣在其他方面。事实上，让我在面试中失败的确定方式就是让我写出一个稍微复杂的 SQL 查询。

我坚信，对于大型系统，不应该允许开发者设计数据模型、设置数据库，也不应该让他们编写 DDL。我过去见过许多实例，其中系统的数据库组件由非数据库专家设计，性能都非常差。慢查询、没有索引、锁竞争等。我参与过的最好的项目都有一个专门的人负责数据库。如果我在领导一个大型项目，我总是会将有经验的 db 专家作为项目的一部分。

假设我们想为我们的架构团队招聘一个数据库专家。他们会有哪些职责？

1) 为团队提供最佳实践建议。

2) 提出 DDL 编码标准。

3) 审查现有系统并提供性能改进指导。

4) 了解竞争格局（Sybase 对 SQL Server 对 Oracle）并影响企业标准。

5) 精通 OLAP、MDX、分析服务等。

6) 知道如何调整数据库和硬件以提供最佳性能。

7) 为所有 DBAs 提供建议。

8) 监控即将到来的技术，如复杂事件处理、时间序列数据库等。熟悉 KDB+、StreamBase、Vhayu 等。

数据库架构师是一个全职工作，我认为所有架构组都应该有一个职位。
