## RX, CCR, CEP, LinqLite, Coreinfo and More

离开几天后，我花了几小时来赶上邮件和博客的更新。

+   .NET 内部[Rx](http://channel9.msdn.com/shows/Going+Deep/Kim-Hamilton-and-Wes-Dyer-Inside-NET-Rx-and-IObservableIObserver-in-the-BCL-VS-2010/)以及 BCL 中的 IObservable/IObserver——如果你想知道他们在[RX](http://codebetter.com/blogs/matthew.podwysocki/archive/2009/10/14/introducing-the-reactive-framework-part-i.aspx)框架中如何实现“Select”函数的类型投射功能，这是一个值得一看的内容

+   我们已经有一段时间没有在 C9 见到[George Chrysanthakopoulos](http://channel9.msdn.com/tags/George+Chrysanthakopoulos/)了。想知道他是否还在[CCR](http://channel9.msdn.com/shows/Going+Deep/CCR-Programming-Jeffrey-Richter-and-George-Chrysanthakopoulos/)领域吗？

+   我看到[Rob](http://blogs.windowsclient.net/rob_relyea/archive/2009/05/19/xaml2009-and-system-xaml-dll-preview-in-net-fx-4-beta1.aspx)在 PDC09 有一个有趣的[会议](http://microsoftpdc.com/Sessions/CL24)。我想那将是关于 System.Xaml.dll 的会议。希望 Rob 尽快发布他的[XAML 分析工具](http://blogs.windowsclient.net/rob_relyea/archive/2009/02/13/avoid-having-default-values-in-xaml-to-do-less-during-load.aspx)，因为 2009 年 2 月已经过去很长时间了😦

+   我看到 Forrester 在 2009 年 8 月发布了“复杂事件处理（CEP）平台，第 3 季度”[报告](http://www.forrester.com/rb/Research/wave%26trade%3B_complex_event_processing_cep_platforms%2C_q3/q/id/48084/t/2)。Aleri 在报告中表现不错，正如预期的那样。我不得不说，“Forrester 认为微软和 SAP 将不会落后于将 CEP 引入他们的产品组合”这一观点很有趣，因为微软在当时已经走上了 CEP 开发的道路，而这份报告是在报告发布时就已经成文了。

+   Infragistics 发布了一篇关于“Infragistics® xamWebGrid for Microsoft® Silverlight 性能”的白皮书。[论文](http://download.infragistics.com/marketing/infragistics-silverlight-xamwebgrid-performance-whitepaper.pdf)中的性能表现看起来不错，但我怀疑它的性能是否与 Syncfusion 的单元格[网格](http://www.syncfusion.com/products/silverlight/grid)处于同一水平。

+   jQuery [示例](http://sixrevisions.com/javascript/10-awesome-techniques-and-examples-of-animation-with-jquery/)——印象深刻，但这是不是避免使用 Adobe/Microsoft RIA 技术的理由？

+   [WPF](http://blogs.msdn.com/llobo/archive/2009/10/29/new-wpf-features-key-gesture-binding.aspx) [4](http://weblogs.asp.net/scottgu/archive/2009/10/26/wpf-4-vs-2010-and-net-4-0-series.aspx) – 继 Rob 上面的帖子之后，新的 XAML/BAML 解析引擎 – 使用 System.Xaml.dll 中提供的新低级 API，分析和管理 XAML 的工具也将更容易创建。讨论 WPF 4 功能的更多内容[在这里](http://blogs.msdn.com/llobo/archive/tags/New+WPF+4+features/default.aspx)。

+   [LinqLite](http://blogs.msdn.com/jasonall/archive/2009/10/27/linqlite.aspx)

+   David Chappell 关于[云计算](http://brunoblogfiles.com/pdfs/The%20Windows%20Azure%20Platform--A%20Perspective--Chappell.pdf)的演讲

+   Silverlight 的 WriteableBitmap[类](http://www.wintellect.com/CS/blogs/jprosise/archive/2009/10/29/more-fun-with-silverlight-3-s-writeablebitmap.aspx)

+   [Coreinfo](http://technet.microsoft.com/en-us/sysinternals/cc835722.aspx) – 在 Windows 7 (x64)和 Windows Server 2008 R2 上发现非均匀内存访问（NUMA）节点、处理器组、插槽、缓存和处理器配置信息

+   在 2009 年的[PDC](http://www.sellsbrothers.com/news/showTopic.aspx?ixTopic=2305)上，“Oslo” sessions 可能是值得一看的，考虑到微软对 Oslo 的投资。

+   GemFire – 数据[管理](http://www.gemstone.com/hardest-problems)
