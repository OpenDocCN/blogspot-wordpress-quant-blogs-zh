## 实时数据精炼

首先，我必须感谢[Hortonworks](http://hortonworks.com/blog/storm-kafka-together-real-time-data-refinery/)为“数据精炼”这个很酷的数据术语。Hortonworks 的“Storm 和 Kafka 一起：实时数据精炼”[文章](http://hortonworks.com/blog/storm-kafka-together-real-time-data-refinery/)为数据处理提供了很好的概述，并解释了为什么 Storm 和 Kafka 能如此好的配合：

> Apache Storm 是一个分布式实时计算引擎，它可靠地处理非边界数据流。尽管 Storm 可以大规模处理数据流，但 Apache Kafka 处理消息同样可以实现大规模。Kafka 是一个分布式发布-订阅实时消息系统，提供了强大的持久性和容错性保证。

教程可在此处找到：[`hortonworks.com/hadoop-tutorial/ingesting-processing-real-time-events-apache-storm/`](http://hortonworks.com/hadoop-tutorial/ingesting-processing-real-time-events-apache-storm/) ，以便深入了解 Storm 和 Kafka 的使用。

同样地，《Putting Apache Kafka To Use: A Practical Guide to Building a Stream Data Platform》提供了进一步的阅读材料。[`www.confluent.io/blog/stream-data-platform-1/`](http://www.confluent.io/blog/stream-data-platform-1/)

最后，《Real Time Streaming with Apache Storm and Apache Kafka》提供了经典的 Twitter 流情感分析。[`zdatainc.com/2014/07/real-time-streaming-apache-storm-apache-kafka/`](http://zdatainc.com/2014/07/real-time-streaming-apache-storm-apache-kafka/)
