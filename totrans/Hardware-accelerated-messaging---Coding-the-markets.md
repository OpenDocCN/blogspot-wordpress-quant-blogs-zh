## Hardware accelerated messaging

### June 21, 2008

Marc asked for [hardware accelerated messaging ideas](http://magmasystems.blogspot.com/2008/06/tibco-and-hardware-acceleration.html). Here’s one: how about using [SCRAMNet](http://www.cwcembedded.com/products/6/1/65.html) to eliminate IP networking and achieve sub microsecond latency ?  You’d need all of your exchange connectivity, pricing and order management processes on boxes in the same machine room linked by a SCRAM fibre loop. But it would go mighty quick. SCRAM pricing is surprisingly reasonable too…
