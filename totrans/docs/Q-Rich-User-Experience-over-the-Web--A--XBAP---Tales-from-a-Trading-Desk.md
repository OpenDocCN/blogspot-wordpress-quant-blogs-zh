## Q.Rich User Experience over the Web? A. XBAP

Tim [blogs](http://blogs.msdn.com/tims/archive/2007/01/30/great-wpf-applications-1-british-library-turning-the-pages.aspx) about the [British Library](http://www.turningthepages.com/) WPF (XBAP) application – XAML Browser Applications (XBAP) are full featured WPF applications.  [WPF Wonderland](http://wpfwonderland.wordpress.com/2007/01/29/creating-product-demos-with-wpf-and-xbap/) notes that XBAP does [restrict](http://wpfwonderland.wordpress.com/2007/01/28/comparing-wpf-applications-and-xbap-whats-the-difference/) the WPF feature set, but XBAP is definately the way to get a visually rich WPF solution deployed via a browser.  MSDN has the full feature set of XBAP [here](http://msdn2.microsoft.com/en-us/library/aa480229.aspx). The British Library used XBAP over [WPF/E](http://blogs.msdn.com/wpfedevcon/archive/2007/01/02/using-opacity-with-a-wpfe-virtual-earth-mashup.aspx) for a reason – but at the expense of cross-platform, cross browser.

Ian Moulster [lists](http://blogs.msdn.com/ianm/archive/2006/02/15/532401.aspx) the other possible solution for rich client experiences – Atlas, [WPF/E](http://blogs.msdn.com/publicsector/archive/2007/01/16/more-wpfe-virtual-earth-and-html-goodness.aspx) and ASP.NET.  As Ian nicely states, before chosing a technology you need to resolve the Reach vs Fidelity issue.  [Flash](http://thewpfblog.com/?p=17) is another consideration.

Tim’s blog also answers the question about the difference between [WPF on XP and Vista.](http://blogs.msdn.com/tims/archive/2007/01/05/comparing-wpf-on-windows-vista-v-windows-xp.aspx)
